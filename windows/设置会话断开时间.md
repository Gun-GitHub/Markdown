要设置 **Windows 用户会话断开时间（即闲置一段时间后自动断开或注销）**，可以有不同的方式，具体取决于你要实现的场景：

## 🎯 常见需求类型：

|需求类型|适用场景|
|--|--|
|❌ 自动注销用户会话|用户空闲 X 分钟后自动注销|
|🔒 自动锁屏|用户空闲 X 分钟后自动锁屏|
|🔌 RDP 会话断开（远程桌面）|用户断开 RDP 一段时间后自动注销或断开|
|🧑‍💻 本地会话断开|本地控制台会话空闲后强制注销或断开|

## ✅ 方法一：设置空闲注销策略（使用组策略）

适用于：**本地登录用户或远程桌面用户空闲自动注销**

### 步骤：

1. 打开 `gpedit.msc`（组策略编辑器）：
   - `Win + R` → 输入：`gpedit.msc` → 回车
2. 定位到以下路径：
   ```
   计算机配置 → 管理模板 → Windows 组件 → 远程桌面服务 → 远程桌面会话主机 → 会话时间限制
   ```
3. 配置以下策略：

|策略名称|建议值|
|--|--|
|空闲会话限制时间（Set time limit for idle sessions）|已启用 → 设置时间（如 15 分钟）|
|断开会话的时间限制（Set time limit for disconnected sessions）|如 5 分钟|
|会话断开时是否自动注销（Terminate session when time limits are reached）|启用|

4. 应用策略：
   - 执行 `gpupdate /force` 使其立即生效。

## ✅ 方法二：使用注册表设置会话空闲限制

适用于没有 `gpedit.msc` 的家庭版用户。

### 注册表路径：

```
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp
```

### 设置键值：

|名称|类型|说明|
|--|--|--|
|`MaxIdleTime`|REG\_DWORD|空闲会话时间（毫秒），如 15 分钟为 `0x00dbba00`|
|`MaxDisconnectionTime`|REG\_DWORD|断开会话最大持续时间|

> ⚠️ 单位是毫秒（1 分钟 = 60000 毫秒）

例如设置 15 分钟空闲断开，可用 `.reg` 文件如下：

```reg
Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp]
"MaxIdleTime"=dword:00dbba00
```

## ✅ 方法三：使用 `tsconfig.msc`（旧版 Server 系统）

- 打开 `tsconfig.msc` → 选择 RDP-Tcp → 设置会话限制。

## ✅ 方法四：使用 `PowerShell` 设置 RDP 会话时间限制

```powershell
# 设置最大空闲时间为 15 分钟
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Terminal Server\WinStations\RDP-Tcp" -Name MaxIdleTime -Value 900000
```

## ✅ 方法五：设置屏幕保护程序超时 + 注销

适用于本地登录用户自动注销：

1. 右键桌面 → 个性化 → 锁屏界面 → 屏幕保护程序设置。
2. 勾选“在恢复时显示登录屏幕”，并选择 “注销” 类型的屏保（或写一个注销脚本）。
3. 设置等待时间，例如 10 分钟。
