 C++ 中静态库（Static Library）和动态库（Dynamic Library，也称共享库）的区别：

### 🔹 静态库（Static Library）

- **链接时机**：在**编译期**或**链接期**被编译进目标程序。
- **文件扩展名**：
  - Windows：`.lib`
  - Linux/Unix：`.a`
- **结果**：最终生成的可执行文件中**包含了库的代码**，文件体积会较大。
- **优点**：
  - 发布和运行时**不依赖外部库文件**。
  - 程序运行时性能略好（无需动态加载）。
- **缺点**：
  - 占用空间更大（每个可执行文件都包含库的副本）。
  - 库更新后，需要**重新编译所有依赖它的程序**。

### 🔹 动态库（Dynamic Library / Shared Library）

- **链接时机**：在**运行时**由操作系统动态加载。
- **文件扩展名**：
  - Windows：`.dll`
  - Linux：`.so`
  - macOS：`.dylib`
- **结果**：可执行文件**不包含库的实际实现**，只包含引用信息。
- **优点**：
  - 节省空间：多个程序可以**共享同一个库文件**。
  - 易于升级：只需**替换库文件**，无需重新编译主程序（只要接口不变）。
- **缺点**：
  - 程序在运行时**依赖动态库的存在**，缺失可能导致启动失败。
  - 稍有性能损耗（运行时加载、符号解析）。

### 举例说明：

假设你有一个函数 `int add(int a, int b);` 存在于一个库中：

- 使用**静态库**：该函数的实现会在编译后被复制进最终的 `.exe` 或 `.out` 文件中。
- 使用**动态库**：最终可执行文件中只有一个指向 `.dll` 或 `.so` 的引用。程序运行时加载这个库来调用 `add`。
