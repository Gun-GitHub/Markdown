# 交换机字段解析

## 系统状态

### sysUpTime        (oid: 1.3.6.1.2.1.1.3)

```
交换机启动到现在工作时长
单位(0.001s<华为交换机>,0.0001s<思科交换机>)
```

### snmp_scrape_duration_seconds

```
snmp 请求响应时间
```

### snmp_scrape_packets_retried

```
指的是在SNMP数据采集过程中，由于某种原因（如网络延迟、目标设备响应超时等）导致请求未能成功完成，随后进行了重试的SNMP数据包数量。
```

### snmp_scrape_packets_sent

```
在SNMP数据采集过程中，由监控系统（如Prometheus的exporter）发送到目标设备（如网络设备、服务器等）的SNMP数据包数量。这些数据包通常包含了对目标设备状态的查询请求。
```

### snmp_scrape_pdus_returned

```
目标设备（如网络设备、服务器等）响应给监控系统（Prometheus的exporter）的SNMP PDU数量。这些PDU通常包含了对请求数据的响应信息。
```

### snmp_scrape_walk_duration_seconds

```
从发起SNMPWALK命令开始，到完成指定MIB分支信息的提取并返回结果所花费的总时间（以秒为单位）。这个时间包括了网络延迟、目标设备处理请求的时间以及数据在监控系统中的处理时间等。
```

## 接口状态

### ifAdminStatus    (oid: 1.3.6.1.2.1.2.2.1.7)

```
字段表示网络接口的管理状态
可选字段:1,2,3
1. (up)：接口被管理员设置为启用状态。这意味着管理员希望接口是活动的，可以处理数据包的发送和接收。在这个状态下，接口应该是运行的，但这并不保证它的实际状态（即 ifOperStatus）确实是运行中的。
2. (down)：接口被管理员设置为禁用状态。这意味着管理员不希望接口进行数据传输，接口在管理上被关闭。即使物理上接口是连接的，在这种状态下，接口也不会处理任何数据。
3. (testing)：接口被设置为测试状态。这意味着接口处于某种测试模式中，可能用于诊断或其他特定目的。在这个状态下，接口通常不会处理正常的网络流量。
```

### ifConnectorPresent    (oid: 1.3.6.1.2.1.31.1.1.1.17)

```
表示接口是否具有物理连接器的一个字段
可选字段:1,2
1. (true)：接口具有物理连接器。这意味着该接口可以通过一个物理连接器与其他网络设备或媒体相连。例如，带有 RJ-45 端口的以太网接口就会被标记为 true。
2. (false)：接口没有物理连接器。这通常用于虚拟接口或不需要物理连接器的接口类型，例如逻辑子接口或某些类型的无线接口。
```

### ifCounterDiscontinuityTime    (oid: 1.3.6.1.2.1.31.1.1.1.19)

```
表示某个网络接口计数器最近一次发生不连续变化的时间
```

### ifIndex    (oid: 1.3.6.1.2.1.2.2.1.1)

```
网络管理中用于唯一标识网络接口的数字。它在整个网络系统中具有唯一性，并广泛应用于各种网络设备和网络管理系统中。
设备索引
一个与物理或逻辑接口相关联的唯一标识号，用于在网络管理中唯一标识每个网络接口。
```

### ifLastChange    (oid: 1.3.6.1.2.1.2.2.1.9)

```
示网络接口的状态最后一次发生变化的时间戳。
```

### ifLinkUpDownTrapEnable    (oid: 1.3.6.1.2.1.31.1.1.1.14)

```
用于控制是否在网络接口发生链接状态变化（如链路建立或断开）时发送告警（trap）。
可选字段:1,2
1. enabled（启用）：一个特定的整数值（如1），表示当接口链路状态发生变化时，将发送告警信息。
2. disabled（禁用）：另一个特定的整数值（如2），表示即使接口链路状态发生变化，也不会发送告警信息。
```

### ifMtu    (oid: 1.3.6.1.2.1.2.2.1.4)

```
表示网络接口能够传输的最大数据单元（MTU）的大小，即该接口支持的最大数据包大小（以字节为单位）
单位(字节)
尽量将MTU设置为一个较大的值，以减少数据包的分片和重组，提高网络传输效率。
```

### ifNumber    (oid: 1.3.6.1.2.1.2.1)

```
用于表示网络设备的接口数量。表示网络设备上当前激活的接口总数。
```

### ifOperStatus    (oid: 1.3.6.1.2.1.2.2.1.8)

```
表示网络接口的实际工作状态。
可选字段:1,2,3,4,5,6,7
1. up：接口正在运行，且能够传递数据包。
2. down：接口已关闭，无法传递数据包。
3. testing：接口正在测试中，可能无法传递数据包。
4. unknown：接口的状态未知。
5. dormant：接口处于休眠状态，通常是由于某种外部条件（如接口未连接任何设备）导致的。
6. notPresent：接口不存在，可能是因为硬件故障或未安装。
7. lowerLayerDown：接口下方的网络层（如物理层或数据链路层）未运行，导致接口无法正常工作。
```

### ifPromiscuousMode    (oid: 1.3.6.1.2.1.31.1.1.1.16)

```
用于表示网络接口的混杂模式（Promiscuous Mode）状态。混杂模式是一种特殊的网络接口模式，允许网络接口接收所有经过它的数据包，而不仅仅是那些目的地址为该接口的数据包。
可选字段:1,2
1. (true)   启用
2. (false)  禁用

性能影响：
    启用混杂模式可能会增加网络接口的负载，因为需要处理更多的数据包。这可能会影响网络性能，特别是在高流量环境中。
安全风险：
    混杂模式允许网络接口接收所有数据包，包括那些可能包含敏感信息的数据包。因此，在启用混杂模式时需要谨慎考虑安全风险，并采取适当的安全措施来保护捕获的数据包不被未经授权的人员访问。
```

## 接口类型

### ifType_info    (oid: 1.3.6.1.2.1.2.2.1.3)

```
用于标识网络接口的类型。这个对象对于网络管理员来说非常重要，因为它提供了关于网络接口物理特性和功能的详细信息。它定义了接口的物理层和数据链路层特性，如以太网、令牌环、FDDI等。

常见的示例：
    ethernetCsmacd（6）：表示以太网CSMA/CD接口。
    iso88023Csmacd（7）：表示ISO 8802-3 CSMA/CD接口。
    fastEther（19）：表示快速以太网接口。
    gigabitEthernet（1000baseLX）：表示千兆以太网接口（使用长波长激光）。
    propVirtual: 这个值通常不是一个标准的SNMP MIB ifType 定义。在标准的SNMP MIB中，并没有直接定义 propVirtual 作为 ifType 的一个值。它可能是某个特定厂商或软件为了表示某种虚拟或专有网络接口而自定义的值。因此，具体的含义需要参考该厂商或软件的文档。
    other(1):通常用于表示一个未知或未定义的接口类型。当SNMP代理（Agent）无法将网络接口归类为任何已知的 ifType 时，它可能会使用 other 值。
    ieee8023adLag:通常表示一个使用IEEE 802.3ad标准的链路聚合组（Link Aggregation Group, LAG）接口。IEEE 802.3ad 是一种链路聚合控制协议，它允许将多个物理接口绑定成一个逻辑接口，以提高带宽、可靠性和弹性。在SNMP MIB中，虽然可能没有直接命名为 ieee8023adLag 的 ifType 值，但链路聚合可能会通过其他方式（如多个物理接口的组合）来表示，或者在某些情况下，厂商可能会扩展MIB以包括特定的 ifType 值来表示链路聚合接口。
```

### ifSpecific    (oid: 1.3.6.1.2.1.2.2.1.22)

```
获取关于网络接口所使用的物理介质（如以太网、光纤等）的详细信息和特定属性。

当ifSpecific对象的值为"0.0"时，这通常表示该网络接口没有与之关联的特定媒体MIB定义，或者该接口的类型不属于任何已知的、需要特定MIB来进一步描述的媒体类型。
```

## 接口输入

### ifHCInBroadcastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.9)

```
表示接口接收到的广播报文总数
单位(个)
这个节点是64位计数器

与 ifInBroadcastPkts 区别:
    该节点是ifInBroadcastPkts的扩充版本，其数据范围更大，使用64位整数（即counter64类型）来表示接收到的广播报文数量。这使得ifHCInBroadcastPkts能够处理更大的数据量，适用于高吞吐量的网络环境。
```

### ifInBroadcastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.3)

```
表示接口上接收到的广播报文的总数。这些广播报文可能包括ARP（地址解析协议）请求、DHCP（动态主机配置协议）发现消息等。

与 ifHCInBroadcastPkts 区别:
    该节点表示接收到的广播报文数量，但其数据范围通常受限于INTEGER类型，即通常为32位整数，最大值为4294967295（即2^32-1）。这意味着当接收到的广播报文数量超过这个值时，计数器可能会回绕或重置。
```

### ifInMulticastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.2)

```
用于表示网络接口上接收到的组播报文个数。对于MAC层协议来说，组播地址包含了组地址和功能地址，这些地址允许将报文发送给网络中的一组设备，而不是单一设备。
该计数器是累加性的，即它会持续记录从网络接口接收到的组播报文数量，直到设备重启或计数器重置为止。
```

### ifInDiscards    (oid: 1.3.6.1.2.1.2.2.1.13)

```
表示网络接口上被丢弃的输入报文数量。尽管这些报文被丢弃，但它们并不一定是由于错误导致的，而是可能因为设备认为这些报文不应该由它处理或者是需要上送CPU进行特殊处理的不合理报文。

与ifInErrors的区别:
    ifInErrors表示由于物理层或链路层错误而导致无法接收的报文数量，这些错误通常是由于硬件故障、线路问题或物理层协议不匹配等原因引起的。而ifInDiscards则更多地反映了网络层或更高层次的决策导致的报文丢弃。
```

### ifInErrors    (oid: 1.3.6.1.2.1.2.2.1.14)

```
表示网络接口上流入的错误报文数量。这些错误报文通常是由于物理层或链路层的问题（如线路噪声、信号衰减、帧校验序列错误等）导致无法被正确接收和处理的报文。
单位(个)
```

### ifHCInUcastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.7)

```
记录的是网络设备（如交换机、路由器等）接口自启动或重置以来接收到的所有单播报文的总数
单位(个)
这个节点是64位计数器
```

### ifHCInMulticastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.8)

```
表示接口接收到的组播报文总数
单位(个)
这个节点是64位计数器
```

### ifHCInOctets    (oid: 1.3.6.1.2.1.31.1.1.1.6)

```
网络设备（如交换机）接口自启动或重置以来接收到的所有输入字节的总和。
单位通常是字节（Byte）
将字节数乘以8（因为1字节等于8比特），然后根据时间差计算出每秒的平均比特率。
```

### ifInNUcastPkts    (oid: 1.3.6.1.2.1.2.2.1.12)

```
表示网络设备接口接收到的非单播数据包的数量。
单位(个)

非单播数据包：
    非单播数据包是指不是直接发送给特定接收者的数据包，包括广播数据包（发送给网络上的所有设备）和多播数据包（发送给网络上的一个特定组）。
ifInNUcastPkts：
    该OID用于记录接口接收到的非单播数据包的总数，包括广播和多播数据包。
```

### ifInOctets    (oid: 1.3.6.1.2.1.2.2.1.10)

```
表示接口（或端口）上接收到的总字节数量。
单位通常是字节（Byte）
```

### ifInUcastPkts    (oid: 1.3.6.1.2.1.2.2.1.11)

```
表示网络设备接口接收到的单播数据包的总数。
单位(个)
```

### ifInUnknownProtos    (oid: 1.3.6.1.2.1.2.2.1.15)

```
表示从网络设备接口接收到的未知协议数据包的数量。
反映了那些因为协议无法识别或不被该网络设备支持而被接收到的数据包总数。
协议兼容性评估
```

## 接口输出

### ifHCOutBroadcastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.13)

```
它代表了接口发送的广播报文总数，包括被丢弃的报文或没有送出的报文。
单位(个)
```

### ifOutBroadcastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.5)

```

与 ifHCOutBroadcastPkts 相比:
                    ifOutBroadcastPkts  ifHCOutBroadcastPkts
```

### ifHCOutMulticastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.12)

```
它代表了接口发送的组播报文总数，包括被丢弃的报文或没有送出的报文。
单位(个)
```

### ifOutMulticastPkts    (oid: 1.3.6.1.2.1.31.1.1.1.4)

```
与 ifHCOutMulticastPkts 相比:
    反正没有 ifHCOutMulticastPkts 精度高,用带 HC 的就对了
```

### ifHCOutOctets    (oid: 1.3.6.1.2.1.31.1.1.1.10)

```
表示网络接口发送的字节总数
单位通常是字节（Byte）
将字节数乘以8（因为1字节等于8比特），然后根据时间差计算出每秒的平均比特率。
```

### ifOutDiscards    (oid: 1.3.6.1.2.1.2.2.1.19)

```
表示由于某种与特定错误条件无关的原因而不能从网络接口发送的报文数量。具体来说，这些被丢弃的报文可能并不是因为传输错误或协议违规，而是由于网络拥塞、队列溢出、报文TTL（生存时间）超时等其他原因导致的。
单位(个)

非错误性丢弃：
    与ifOutErrors不同，ifOutDiscards表示的是由于非错误性原因而被丢弃的报文数量。这些原因可能包括网络拥塞、资源限制或管理策略等。
累计数目：
    ifOutDiscards的值是一个累计数目，表示自网络接口启动以来，由于上述原因而被丢弃的报文总数。
```

### ifOutErrors    (oid: 1.3.6.1.2.1.2.2.1.20)

```
表示网络设备接口在发送数据过程中出现的错误报文数量。具体来说，ifOutErrors记录了由于传输错误而无法成功发送的报文总数。

错误性统计：
    与ifOutDiscards不同，ifOutErrors专注于统计由于传输错误（如校验和错误、硬件故障等）而被丢弃的报文数量。
累计数目：
    ifOutErrors的值是一个累计数目，它表示自网络接口启动以来，由于传输错误而被丢弃的报文总数。
性能评估：
    在网络性能评估中，ifOutErrors的值是一个重要的参考指标。高值的ifOutErrors可能表明网络接口存在传输问题，需要进一步排查和优化。
```

### ifOutNUcastPkts    (oid: 1.3.6.1.2.1.2.2.1.18)

```
它代表了一个网络接口发送的非单播（Non-Unicast）数据包数。非单播数据包主要包括组播（Multicast）和广播（Broadcast）数据包。
单位(个)
ifOutNUcastPkts是一个累计数目，表示从该网络接口发送的组播和广播数据包的总数。这个值在达到其最大值时可能会自动清零，具体行为取决于设备的实现。
```

### ifOutOctets    (oid: 1.3.6.1.2.1.2.2.1.16)

```
表示网络接口发送的字节总数。
单位(字节)
ifOutOctets记录了从网络接口发送的所有字节（包括数据帧的头部和尾部）的总数。这是一个累计数目，表示自网络接口启动以来或自上次计数器清零以来发送的总字节数。
```

### ifOutQLen    (oid: 1.3.6.1.2.1.2.2.1.21)

```
表示网络设备接口上的输出报文队列长度。具体来说，这个值反映了在特定时刻，该网络接口待发送的数据包（或报文）在输出队列中的数量。
通过监控ifOutQLen的值,可以了解网络接口的发送队列状态，进而评估网络接口的发送性能和潜在的拥塞情况。

队列长度：
    ifOutQLen的值表示了队列中待发送数据包的数量。这个值越高，可能意味着发送队列越长，发送延迟也可能越大。
动态变化：
    由于网络流量的动态性，ifOutQLen的值会随着时间的推移而不断变化。因此，在监控时需要关注其变化趋势和峰值情况。
与性能的关系：
    通常情况下，如果ifOutQLen的增长速度持续高于ifOutOctets（接口发送的字节数）的增长速度，这可能表明网络接口存在拥塞或性能瓶颈。
```

```
表示网络设备接口发送的单播数据包数。
单位(个)
```

## 接口速率

### ifHighSpeed    (oid: 1.3.6.1.2.1.31.1.1.1.15)

```
提供接口带宽的估计值，便于网络管理和监控。
单位(1,000,000 bit/s)
这是个定值,只要接口设备确定了,这个值就不会变
```

### ifSpeed    (oid: 1.3.6.1.2.1.2.2.1.5)

```
表示网络接口的传输速率或速度。具体来说，ifSpeed 对象提供了网络接口的理论最大传输速率（以比特每秒，即bps为单位）的信息。
单位(比特每秒，即bps)

与 ifHighSpeed 相比:
    没有 ifHighSpeed 精确,用上面那个
```
