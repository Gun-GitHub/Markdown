## 介绍

SMTP（Simple Mail Transfer Protocol）是一种用于发送电子邮件的通信协议。它通常用于邮件发送服务器之间的邮件传输，负责将邮件从客户端发送到邮件服务器，或者在服务器之间转发邮件。

SMTP 工作流程大致如下：

1. **客户端发送邮件**：邮件客户端（如 Outlook、Thunderbird 或 web 邮件客户端）使用 SMTP 将邮件发送到邮件服务器。
2. **邮件服务器接收邮件**：SMTP 邮件服务器接收到邮件后，会将其传递到目标邮件服务器。
3. **邮件转发**：如果目标邮件服务器不在同一台服务器上，邮件会通过一系列 SMTP 服务器转发，直到到达目标服务器。
4. **目标服务器存储邮件**：目标服务器收到邮件后，会存储邮件，待收件人获取。
5. **收件人读取邮件**：收件人使用 POP3 或 IMAP 协议从邮件服务器上下载或同步邮件。

SMTP 使用的是 **TCP 端口 25**（通常用于服务器间的通信）。不过，由于反垃圾邮件和安全问题，常见的 SMTP 安全端口包括：

- **端口 587**：用于邮件客户端发送邮件（推荐）。
- **端口 465**：用于加密邮件发送（不常见，但曾经用于 SMTPS）。

SMTP 通常不负责接收邮件。接收邮件的协议包括 **POP3**（Post Office Protocol）和 **IMAP**（Internet Message Access Protocol）。
