要想 k8s dns 能够解析 外部域名,需要修改以下文件:

project: system-> namespace: kube-system -> configMap: coredns:

```sh
.:53 {
    errors
    health {
       lameduck 5s
    }
    ready
    kubernetes cluster.local in-addr.arpa ip6.arpa {
       pods insecure
       fallthrough in-addr.arpa ip6.arpa
       ttl 30
    }
    prometheus :9153
    # bind 169.254.25.10  # NodeLocalDNS的IP地址
    forward . 114.114.114.114
    cache 30
    loop
    reload
    loadbalance
}
```

将百度的域名解析服务器地址加进去: 

```sh
forward 114.114.114.114
```

linux 配置 dns 解析器,可以使得 linux 可以解析 k8s 中这种 svc.namespace 域名,配置文件为: /etc/resolv.conf, 内容如下:

```sh
# Generated by NetworkManager
nameserver 169.254.25.10
search default.svc.cluster.local svc.cluster.local cluster.local
options ndots:5
```

字段解析可以查看 linux

在设置完成之后要使用一下指令加锁,否则在系统重启之后会重置 /etc/resolv.conf 文件:

```sh
chattr +i /etc/resolv.conf
```

解锁

```sh
chattr -i /etc/resolv.conf
```
