在 Windows 系统中，如果你想设置一个“用户启动指令”，也就是让某些程序或脚本在用户登录时自动启动，可以通过以下几种方式来实现：

## ✅ 方法一：使用「启动」文件夹（适合普通用户）

这是最简单的方法，把你的启动指令做成一个快捷方式放到启动文件夹中。

### 操作步骤：

1. **打开启动文件夹：**
   - 按下 `Win + R`，输入：
     ```
     shell:startup
     ```
   - 回车后打开的是当前用户的启动文件夹。
2. **创建启动项：**
   - 将你要执行的程序或脚本的快捷方式拖进去即可。
     - 比如 `my_script.bat` 脚本，右键它 → 创建快捷方式 → 拖入启动文件夹。

## ✅ 方法二：使用注册表设置（适合高级用户）

你也可以通过注册表设置开机启动项。

### 操作步骤：

1. **打开注册表编辑器：**
   - `Win + R` → 输入 `regedit` → 回车。
2. **找到路径：**
   ```
   HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
   ```
3. **添加字符串值：**
   - 右键右侧空白处 → 新建 `字符串值（String Value）`
   - 名称任意（比如：`MyApp`），数值设为完整路径，如：
     ```
     "C:\Users\zhoujun\Scripts\my_script.bat"
     ```

> ✅ 此方法不适用于需要管理员权限的程序，适合普通用户程序或脚本。

## ✅ 方法三：使用「计划任务」设置（适合管理员权限的任务）

如果你想在用户登录时执行一些需要权限的脚本，建议用“计划任务”。

### 操作步骤：

**打开任务计划程序：**

- 开始菜单搜索 "任务计划程序" 或运行：
  ```
  taskschd.msc
  ```

创建任务：

- 右侧点击“创建任务”（**不是基本任务**）。
- 设置：
  - 触发器：登录时
  - 操作：启动程序或脚本
  - 权限：可以勾选“使用最高权限运行”

**保存并测试任务。**

## ✅ 方法四：使用组策略（适用于域环境或专业版）

如果你使用的是 Windows 专业版以上，可以使用 `gpedit.msc`。

- 打开组策略编辑器 → `用户配置` → `Windows 设置` → `脚本（登录/注销）`
- 添加你希望在用户登录时执行的 `.bat` 或 `.ps1` 脚本

```powershell
start "" "C:\Program Files\FsScreen\FsScreen.exe"
```

## 📌 常见脚本类型

|类型|建议调用方式|
|--|--|
|`.exe`|直接调用即可|
|`.bat`|放在启动文件夹或注册表中调用|
|`.ps1`|推荐使用 PowerShell 启动脚本，可能需要配置执行策略|

```powershell
powershell -ExecutionPolicy Bypass -File "C:\path\to\script.ps1"
```
